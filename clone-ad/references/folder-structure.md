# Working directory layout

Set this up at the project root (e.g. `~/products/<clone-name>/`) before running any pipeline step. Every recipe assumes this layout.

```
<clone-name>/
├── reference/                       # the source ad and its analysis
│   ├── source.mp4                   # downloaded reference (or symlink to local file)
│   ├── source-url.txt               # the original URL (for record)
│   └── shot-list.json               # Gemini analysis output
├── product/                         # the user's product image
│   ├── product.png                  # generated by Nano Banana 2 (or supplied by user)
│   └── product-prompt.md            # the Nano Banana prompt that produced it
├── clone/                           # the rewritten shot list and final video
│   ├── prompt.md                    # the rewritten Seedance prompt with CUT. SHOT N: language
│   ├── clone.mp4                    # the Seedance output
│   └── verification.json            # Gemini re-analysis of the clone for QA
├── plan.md                          # rolling notes — what's locked, what's pending
└── post.md                          # if the user wants to publish, the Zernio payload (see generate-carousel skill for that flow)
```

## Folder responsibilities

- `reference/` — Source-of-truth artifacts about the ad being cloned. Keep `shot-list.json` after analysis so you can rewrite without re-paying for Gemini analysis credits.
- `product/` — The hero product image that Seedance uses as the i2v starting frame. Keep the prompt that produced it so iterations are reproducible.
- `clone/` — The rewritten Seedance prompt, the rendered clip, and the verification analysis. The prompt file is the source of truth for any regeneration.
- `plan.md` — Rolling notes on the cloning decisions: what reference, what dialogue rewrite, what wardrobe/setting variations, what failed.
- `post.md` — Optional. If publishing, snapshot the publisher payload here (account id, caption, schedule).

## Naming conventions

- Reference shortname (slug) is derived from the brand or topic: `gymshark-leg-day`, `chocolate-mousse-asmr`, `claude-code-tutorial`. Use it in the working-dir name and inside file headers.
- Output MP4 is always `clone/clone.mp4`. Don't version inside the filename — version by branching the working directory or by saving to `clone/clone-v2.mp4` only when you have a reason to keep both.
- Verification JSON is named `verification.json` (one per clone iteration). Old iterations land at `verification-v1.json` etc.

## What to commit / share

- Do commit `prompt.md`, `shot-list.json`, `verification.json` — these capture intent and outcomes.
- Do NOT commit the downloaded reference video itself if it's public source content; reference it by URL in `source-url.txt` instead.
- Do NOT commit secrets — the env file lives outside this directory.
