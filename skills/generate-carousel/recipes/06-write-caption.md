# Recipe 06 — Write the caption

## Goal

Produce a publishing-ready caption at `caption.md` that pairs with the carousel and ends in the CTA matching the closer slide.

## Steps

1. Read `references/caption-format.md` for the structure and voice.

2. Open the cover slide image and the closer slide image. The caption's first line mirrors the cover headline; the last line mirrors the closer's CTA.

3. Draft the caption in this order:

   a. **Hook line** — paraphrase or restate the cover headline. Add the `↓` or `→` glyph if the niche uses it.

   b. **One-sentence framing of the cultural moment or problem** — what most people get wrong, what the trend is, what the surprising claim is.

   c. **One-sentence reframe / promise** — what the carousel teaches, in one line.

   d. **Body** — mirror the body slides as a tight list or a short paragraph. Keep each line ≤ 80 characters.

   e. **Stakes line — what happens if the reader skips this.**

   f. **Stakes line — what happens if the reader applies this.**

   g. **CTA** — exact match to the closer slide. If the closer says `Comment "REAL"` then the caption ends with `Comment "REAL" and we'll send you …`.

   h. **Hashtags** — 5–8 hashtags on a separate final line. Mix one branded handle-style tag with topic tags.

4. Length check:
   - Total caption (with hashtags) ≤ 2200 characters
   - Body block ≤ 6 list items
   - No "Did you know" openers, no "In today's post" framing

5. Save to `caption.md` at the project root.

6. Have the user review and approve the caption before any publishing happens.

## Output

```
caption.md
```

## Done when

- Caption file exists and is under 2200 chars
- Hook line mirrors the cover headline
- CTA matches the closer slide exactly
- The user has approved
