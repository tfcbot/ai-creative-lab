# Carousel format grammar

A small number of visual format patterns recur across high-engagement Instagram carousels. Each is a recognizable cover composition + a recognizable inside-slide structure that fans of the format expect.

## Visual constants in every well-built carousel

Before format choice — these never change:

- 4:5 portrait crop (1080×1350 target)
- One human or hero object filling the frame on the cover
- Logo lockup placed top-center or top-left, small, three lines stacked tightly
- Editorial sans-serif headline stacked on multiple lines, layered ON TOP of the photography (not placed beside the subject)
- Recurring single-italic-word motif inside an otherwise sans headline
- A small CTA element near the bottom — pill, arrow, or imperative ("SWIPE →", "SAVE THIS →", "FOLLOW FOR MORE →")
- No emojis on covers, no captions burned in beyond headline + CTA
- Hyper-realistic photography or product shot — natural skin texture, soft warm grade, neutral or sky backdrops

That's the system. The variations sit inside it.

## The 6 repeatable formats

### 1. Tool / option comparison ("X vs Y")
Two subjects head-to-head with a versus motif. Inside slides are stacked pair comparisons — same prompt rendered two ways, with annotation arrows pointing at specific differences.
- Use when: two tools, models, methods, or options compete on a real test case
- Casting: symmetric two-subject portrait, mirrored pose, identical wardrobe
- Closer: a recommendation slide naming a winner with a soft CTA

### 2. Workflow primer ("How to use X for Y")
Single subject photographed candidly with a branded prop tied to the topic. Inside slides are numbered steps, each with a sample output thumbnail.
- Use when: introducing one tool or workflow
- Casting: single subject close-medium, branded prop in hand near the face
- Closer: CTA pill with the resource link

### 3. "This is X" reveal
Ultra-photorealistic close-up that looks 100% real. Big italicized headline naming the surprising claim. Inside slides break down the realism cues, one body part or detail per slide.
- Use when: proving an aesthetic claim, then teaching what creates it
- Casting: high-detail beauty close-up, full-bleed
- Closer: comment-keyword CTA so viewers pull the resource

### 4. Asset / use-case launcher ("Generate AI [X] for your [vertical]")
Single subject styled with a hero product. Inside slides are vertical-specific prompt templates plus 3–6 generated example renders.
- Use when: pitching a vertical (fashion, beverage, beauty, interiors)
- Casting: medium shot, model + hero product, soft window light
- Closer: CTA to a guide or community

### 5. Listicle / number drop ("N [things]")
Cover features a model with a literal physical embodiment of the topic — a printed card, a logo'd object, an oversized number prop. Inside slides are one item per slide with display type and supporting visual.
- Use when: lead-magnet drops, free-resource bundles, comment-bait CTAs
- Casting: model close-medium, branded prop physically integrated into the face/body
- Closer: comment-keyword CTA ("Comment 'X' and get the …")

### 6. Hero product showcase ("The [Object]")
Single iconic product, dead-center, isolated on a clean cyc. Inside slides show the same product applied across formats (1:1 IG / 9:16 Reel / 16:9 banner).
- Use when: product launches, packaging-driven content, design-system showcases
- Casting: isolated product center frame, dramatic single-color accent
- Closer: SWIPE pill or a soft brand handle

## Choosing a format

Match the format to the user's content angle:

- **Comparison content** → Format 1
- **Educational walkthrough** → Format 2
- **Aspirational / aesthetic claim** → Format 3
- **Vertical pitch / use-case demo** → Format 4
- **Lead-magnet drop / curiosity hook** → Format 5
- **Single-product showcase** → Format 6

If two formats fit, pick the one with simpler casting — fewer subjects, fewer props — to reduce regeneration risk on the body slides.

## Slide count by format

- Format 1 (comparison): 5–7 slides — cover + 3–5 comparison slides + closer
- Format 2 (workflow): 6–10 slides — cover + step slides + closer
- Format 3 (reveal): 6–8 slides — cover + 4–6 detail slides + closer
- Format 4 (asset launcher): 7–10 slides — cover + 5–7 templated examples + closer
- Format 5 (listicle): 4–8 slides — cover + N item slides + closer (N = number in the cover headline)
- Format 6 (product): 4–6 slides — cover + 3–5 application slides

Default for a tight first ship: 7 slides — covers, body, closer with a clear CTA.
