# Recipe 07 — Publish

## Goal

Publish or schedule the carousel to Instagram via Zernio. This is the only step that affects external systems — confirm explicitly with the user before submitting.

## Steps

1. Read `references/zernio-instagram.md`.

2. Verify the target account is connected:

   ```
   GET https://api.zernio.com/v1/accounts
   Authorization: Bearer $ZERNIO_API_KEY
   ```

   Filter for `platform === "instagram"` and the user's intended username. Capture `_id` (account id) and `profileId._id` (profile id).

3. Build the post payload as described in `references/zernio-instagram.md`:

   - `content` — the caption from `caption.md`
   - `mediaItems` — ordered list of `{ "type": "image", "url": "<CDN URL>" }`, slide 1 first
   - `platforms[0].platform` — `"instagram"`
   - `platforms[0].accountId` — from step 2
   - `platforms[0].profileId` — from step 2
   - `platforms[0].scheduledFor` — see "Timing" below
   - `platforms[0].platformSpecificData.instagramSettings.mediaType` — `"CAROUSEL_ALBUM"`
   - `platforms[0].platformSpecificData.instagramSettings.shareToFeed` — `true`
   - `hashtags` — array of plain strings (no `#`)

4. **Surface the full payload in chat** for the user to review. Confirm:
   - Caption text is correct
   - Slide order matches the intended swipe order
   - Target account username is correct
   - Schedule timing matches the user's intent

   Wait for explicit approval. Do not submit before approval.

5. **Timing options:**

   - **Publish now** — set `scheduledFor` to ~60 seconds in the future (`new Date(Date.now() + 60000).toISOString()`). Setting it to "now" can race the media-items save; 60s out is the safe default.
   - **Schedule for later** — set `scheduledFor` to the requested ISO timestamp.
   - **Drafts mode** — set `scheduledFor` far in the future (e.g. 1 year out). Zernio holds it without publishing; the user can publish manually later from the Zernio app or, on supported plans, from the Instagram native drafts inbox.

6. Submit:

   ```
   POST https://api.zernio.com/v1/posts
   Authorization: Bearer $ZERNIO_API_KEY
   Content-Type: application/json
   <payload>
   ```

   Expect HTTP 201 with the created post object including `_id`.

7. Poll the post's status until it reports `published` or surfaces a `publishError`:

   ```
   GET https://api.zernio.com/v1/posts?id=<post_id>
   ```

   - `status: "publishing"` → still in flight, wait ~30s
   - `status: "published"` → submitted to Instagram successfully (the public URL on Instagram's side may take 1–5 more minutes to appear)
   - `status: "failed"` → check `platforms[0].platformSpecificData.publishError` and remediate

8. Save the final post `_id`, the IG `pendingContainerId`, and the Zernio internal status to `post.md` for record.

## Confirmation requirements (load-bearing)

- Posting to a public account is high-blast-radius. The skill MUST confirm with the user before submitting `POST /v1/posts`.
- Surface the full payload in chat. Wait for explicit approval. Do not interpret a previous "go ahead" earlier in the session as standing approval for this specific submission.

## Common failure modes

- **Image fetch failure on Zernio's side** — your CDN URL isn't publicly reachable, or it requires auth. Re-host on a public CDN per `references/vidjutsu-cdn.md`.
- **Caption too long** — cap at 2200 characters including hashtags.
- **`mediaType` mismatch** — must be `CAROUSEL_ALBUM` for image carousels. `REELS` or `IMAGE` will fail.
- **Account not active** — `/v1/accounts` returned `isActive: false`. The user needs to reconnect in the Zernio app.

## Output

```
post.md   (final payload + Zernio post _id + IG container id)
```

## Done when

- Zernio reports `status: "published"`
- The user can verify the carousel is live on their Instagram profile
- `post.md` is updated with the post id and container id for record

## Pipeline complete

You've shipped a research-driven, format-aware Instagram carousel — every slide generated by gpt-image-2 with typography baked in, hosted on a stable CDN, captioned, and published via Zernio. The entire `scenes/` and `caption.md` set is reusable for variant carousels in the same brand voice — change the body slides, keep the cover and closer system, generate a new piece in under an hour.
